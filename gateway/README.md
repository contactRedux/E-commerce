# API Gateway

The single entry point for all client traffic in the e-commerce platform. Built with **Spring Cloud Gateway** (reactive / WebFlux), it handles JWT validation, rate limiting, CORS, HTTPS termination, and transparent routing to six downstream microservices.

---

## Architecture Role

```
Client (HTTPS)
    │
    ▼
┌──────────────────────────────────────────────────┐
│                   API Gateway :8080               │
│  1. RequestLoggingGatewayFilter  (order -2)       │
│     - Assigns X-Request-Id                       │
│     - Logs method / path / status / duration      │
│  2. JwtAuthenticationGatewayFilter  (order -1)    │
│     - Validates RS256 JWT (public key only)       │
│     - Forwards X-User-Id, X-User-Role headers     │
│     - Strips Authorization header                 │
│  3. Spring Cloud Gateway routing                  │
│     - Redis-backed rate limiting per user / IP    │
│     - Rewrites paths and proxies to services      │
└──────────────────────────────────────────────────┘
    │          │         │        │       │       │
  user      product    cart    order  payment  notification
 :8081       :8082     :8083   :8084   :8085     :8086
```

## JWT Validation

- The gateway validates tokens using **RS256** (RSA asymmetric signature).
- The **RSA private key lives exclusively in the User Service** — the gateway only ever holds the public key.
- On a valid token the gateway:
  1. Extracts `sub`, `role`, and `email` claims.
  2. Adds `X-User-Id`, `X-User-Role`, and `X-User-Email` headers to the downstream request.
  3. **Removes the `Authorization` header** so the raw token is never forwarded.
- On an invalid / expired token the gateway immediately returns `401 Unauthorized`.
- Token content is **never logged**.

### Public routes (bypass JWT)

| Path prefix              | Reason                              |
|--------------------------|-------------------------------------|
| `/auth/register`         | User registration (no token yet)    |
| `/auth/login`            | Login (no token yet)                |
| `/auth/refresh`          | Token refresh                       |
| `/auth/public-key`       | RSA public key endpoint             |
| `/payments/webhook`      | Stripe webhook (Stripe signature)   |
| `/actuator`              | Internal health / metrics           |
| `/docs`                  | API documentation (static HTML)     |

## Rate Limiting

Rate limiting is implemented with the Spring Cloud Gateway **`RequestRateLimiter`** filter backed by Redis (token-bucket algorithm).

| Parameter        | Value  | Description                          |
|------------------|--------|--------------------------------------|
| `replenishRate`  | 10     | Sustained requests per second        |
| `burstCapacity`  | 20     | Maximum tokens in the bucket         |

The **key resolver** keys by `X-User-Id` header for authenticated requests, and falls back to the client IP address for public (unauthenticated) routes.

## CORS Configuration

Allowed origins are read from the `ALLOWED_ORIGINS` environment variable (comma-separated). The `CorsWebFilter` bean applies globally to all routes. Credentials are allowed; max preflight age is 3600 seconds.

```
ALLOWED_ORIGINS=https://myapp.example.com,https://admin.example.com
```

The `DedupeResponseHeader` default filter prevents duplicate `Access-Control-*` headers when both the gateway and a downstream service add them.

## HTTPS / TLS

- TLS terminates at the gateway; downstream services communicate over HTTP on the internal Docker network.
- The keystore is a PKCS12 file. In local development it is generated by `infrastructure/certs/generate-certs.sh`.
- To disable SSL for quick local testing (without a keystore): `SSL_ENABLED=false`.

```bash
# Generate the local self-signed keystore
bash infrastructure/certs/generate-certs.sh
```

## Route Table

| Gateway Path      | Upstream Service             | Notes                        |
|-------------------|------------------------------|------------------------------|
| `/auth/**`        | `user-service:8081`          | Public routes bypass JWT     |
| `/users/**`       | `user-service:8081`          | Requires JWT                 |
| `/products/**`    | `product-service:8082`       | Requires JWT (write ops)     |
| `/categories/**`  | `product-service:8082`       | Requires JWT (write ops)     |
| `/cart/**`        | `cart-service:8083`          | Requires JWT                 |
| `/orders/**`      | `order-service:8084`         | Requires JWT                 |
| `/payments/**`    | `payment-service:8085`       | JWT except `/webhook`        |
| `/actuator/**`    | Gateway (local)              | Public                       |
| `/docs/**`        | Gateway (static HTML)        | Public                       |

Routes are defined programmatically in [`GatewayRoutesConfig.java`](src/main/java/com/ecommerce/gateway/config/GatewayRoutesConfig.java).

## Running Locally

### Via Docker Compose (recommended)

```bash
# From the repository root
docker compose up --build gateway
```

### Standalone

```bash
cd gateway
mvn spring-boot:run \
  -Dspring-boot.run.jvmArguments="-DSSL_ENABLED=false -DJWT_PUBLIC_KEY="
```

---

## Running Tests

```bash
cd gateway
mvn test
```

Coverage report (JaCoCo) is generated at `target/site/jacoco/index.html` after `mvn verify`.
