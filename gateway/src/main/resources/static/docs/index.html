E-Commerce Platform — API Reference

All requests flow through the API Gateway on port 8080 (HTTP) / 8443 (HTTPS).

Authentication: All endpoints require an Authorization: Bearer <token> header unless marked as public. Tokens are issued by the User Service (POST /auth/login).

Services

User Service

:8081 public: /auth/**

User registration, login, JWT issuance (RS256), and profile management.

Product Catalog Service

:8082

Products, categories, inventory management, search, filtering, and pagination (MongoDB).

Shopping Cart Service

:8083

Per-user cart management with Redis persistence and 7-day TTL.

OpenAPI Spec ↗

Order Service

:8084

Order placement, lifecycle management (PENDING → CONFIRMED → SHIPPED → DELIVERED), and history.

OpenAPI Spec ↗

Payment Service

:8085 public: /payments/webhook

Stripe payment intent creation, webhook handling (Stripe signature verification), and refunds.

OpenAPI Spec ↗

Notification Service

:8086 consumer-only

Event-driven email (SendGrid) and SMS (Twilio) notifications. No public REST API — Kafka consumer only.
